# 同博主固定词块复习原则稿

## 1. 这份稿件解决什么问题

当前系统已经具备以下能力：

- 能把上传内容切成可训练的句子片段
- 能围绕单句做训练、复习和难度控制
- 能逐步积累用户的词汇掌握和训练进度

但系统仍然缺少一个关键能力：

**当同一个博主不断上传新内容时，系统如何把这些内容组织成可持续强化的训练材料，而不是一堆彼此孤立的句子。**

如果正面去做一个完整的“博主发音画像系统”，实现成本会很高：

- 需要长期稳定地识别同一个博主
- 需要持续积累发音、连读、弱读、语速等统计
- 需要依赖较重的语音分析和额外的推理能力
- 第一版很难快速验证产品价值

因此，我们需要一个更轻、更稳、更接近训练本质的切入点。


## 2. 核心重定义

我们不先尝试“理解这个博主的全部发音规律”。

我们先做一件更可计算、更贴近训练价值的事情：

**抓住同一个博主反复出现的连续词块，并围绕这些词块组织复习。**

这意味着：

- 我们不先以“整条视频”为核心
- 我们不先以“整句语义相似”为核心
- 我们先以“同博主反复说出的多词连续片段”为核心

例如：

- `i want to`
- `going to be`
- `at the end of`
- `a lot of`
- `what I want to do`

这些连续词块在真实口语里，往往正是连读、弱读、节奏压缩和边界模糊最容易发生的地方。

所以，这个方案不是退而求其次，而是直接抓住了听力训练中最容易失真的部分。


## 3. 产品原则

### 原则一：先利用可重复的语言外形，再逐步逼近更深的发音规律

第一阶段不依赖复杂画像，不依赖大模型，不依赖高成本语音能力。

先利用文本中稳定、可重复、可索引的结构：

- 同一博主
- 连续词块
- 多次复现

这让系统可以先形成一套真实可用的训练闭环。


### 原则二：训练单元优先小于句子，而不是大于句子

传统内容推荐倾向于推荐整条内容、整句或整段。

但在听力训练里，真正造成理解困难的，常常不是整句意义，而是句中某个连续词块的发音方式。

因此，系统优先围绕“词块”而不是“整句”组织复习，会更贴近训练目标。


### 原则三：第一版不做人为限制，先完整保留真实语言分布

第一版不预设哪些词块“有价值”，哪些词块“太泛”。

原因是：

- 在通用 NLP 里看似普通的词块，在听力训练里可能非常关键
- 高频口语块恰恰可能是最容易连读、弱读、听不清的地方
- 过早过滤会把真正有训练价值的样本误删

因此第一版采用：

- 全量提取
- 全量保留
- 轻量排序

而不是：

- 人工过滤
- 语言学门槛
- 经验性剔除


### 原则四：先做“固定词块复习”，再做“相似句推荐”

“固定词块复习”是底层能力，“相似句推荐”是上层编排能力。

先做固定词块复习的原因：

- 实现更简单
- 结果更稳定
- 更容易解释给用户
- 更容易沉淀为系统能力

一旦系统已经知道某个博主有哪些反复出现的连续词块，后续再做相似句推荐就会自然很多。


## 4. 为什么这条路线成立

这条路线成立，不是因为它“简单”，而是因为它准确地命中了训练目标。

听力训练的难点很多时候不是：

- 用户不认识单词
- 用户不理解句意

而是：

- 多个词连在一起之后，听不出边界
- 高频表达在真实语速下变形
- 用户无法快速适应同一个人的稳定说法

而“同博主固定词块复习”恰好能解决这些问题：

- 同一博主重复说同一词块，发音习惯通常更稳定
- 同一词块在不同句子中复现，有利于形成“模式掌握”
- 用户练习的不再是孤立句子，而是一类真实反复出现的说法

这使系统从“句子库”逐步进化成“语言模式库”。


## 5. 第一版能力定义

第一版不追求复杂的博主画像，不追求完美的发音分析。

第一版只做以下事情：

### 5.1 建立同博主维度

系统需要能够识别或标记：

- 哪些内容来自同一个博主

第一版可以接受较轻的实现方式，例如：

- YouTube 元数据识别
- 或者人工指定同一博主
- 或者为内容绑定统一的 `creator_key`


### 5.2 为每个句子提取连续词块

对每个训练片段的文本做标准化处理，然后提取连续的 `3-gram` 和 `4-gram`。

示例：

句子：

`I want to talk about this today`

提取的词块：

- `i want to`
- `want to talk`
- `to talk about`
- `talk about this`
- `about this today`
- `i want to talk`
- `want to talk about`
- `to talk about this`
- `talk about this today`


### 5.3 将词块挂回到同博主的真实句子上

系统需要知道：

- 某个词块出现在哪些句子里
- 这些句子属于哪个博主
- 这些句子来自哪些内容

这样就能形成最基础的“词块 -> 例句集合”映射。


### 5.4 在训练后推荐“这个博主还这样说过”

当用户完成一个句子的训练后，系统不只是结束当前任务，而是基于这个句子的词块，召回同博主中包含相同词块的其他句子。

这样前端可以展示一个很自然的模块：

**这个博主还这样说过**

展示内容可以是：

- 当前句子里的关键词块
- 该词块在同博主中出现过的其他句子
- 用户可以继续点开这些句子做复习


## 6. 这不是普通推荐系统，而是教育型推荐

虽然这套机制看起来像推荐系统，但它优化的目标不是点击率，也不是停留时长。

它优化的是：

- 理解提升速度
- 对同一博主说法的适应速度
- 高频口语块的稳定识别能力
- 挫败感更低的递进式复习体验

因此，这套机制应被理解为：

**围绕同一说话者语言模式的训练编排系统。**


## 7. 第一版明确不做什么

为了保证路线清晰，第一版明确不做以下事情：

- 不依赖大模型做句义相似判断
- 不依赖微软发音评估作为推荐前置条件
- 不强行建立复杂的博主发音画像
- 不预先加入大量人工过滤规则
- 不试图一步做到“最智能推荐”

第一版的目标不是“最聪明”，而是：

**先形成一个低成本、稳定、可解释、可验证的正反馈闭环。**


## 8. 后续演进路线

在“同博主固定词块复习”跑通之后，系统可以自然沿着以下方向增长：

### 阶段 1：固定词块复习

- 同博主
- 提取连续词块
- 召回包含同样词块的句子

这是当前最小可行主线。


### 阶段 2：同博主相似句推荐

在已有词块索引的基础上，进一步寻找：

- 共享多个词块的句子
- 前半句相同、后半句变化的句子
- 围绕同一高频表达展开的句族

这样系统会从“词块复习”升级到“句族复习”。


### 阶段 3：结合用户训练反馈做个性化排序

系统可以根据用户近期记录，优先推荐：

- 用户刚错过的词块
- 用户尚未掌握的句族
- 同一博主中最值得继续强化的模式

这时系统开始具备真正意义上的个性化正反馈能力。


### 阶段 4：再叠加发音现象精排

等语音层数据更成熟之后，再把：

- 连读
- 弱读
- 闪音
- 省略

等能力作为精排加分项，而不是第一版的前置依赖。


## 9. 这条路线的本质价值

这条路线最重要的地方，不在于它“省掉了大模型”，而在于它找到了一个更贴近产品本质的切入方式。

它把一个看似难以直接实现的问题：

**如何持续理解同一个博主，并把这种理解反哺训练系统**

重写成了一个可以立即落地的问题：

**如何围绕同一个博主反复出现的连续词块，组织可重复、可递进、可积累的复习。**

这就是这条方法的创造性所在。


## 10. 一句话定义

**先利用同一博主反复出现的固定词块，建立稳定的复习闭环；再在这个闭环之上，逐步长出相似句推荐、个性化排序和更深层的发音建模能力。**
