# 听力训练系统 — 使用说明

基于**可理解输入 i+1 原则**的自适应英语听力训练系统。

---

## 快速开始

### 1. 配置 API Keys

```bash
cp .env.example .env
```

打开 `.env`，填入：

```
AZURE_SPEECH_KEY=你的Azure语音Key
AZURE_SPEECH_REGION=eastasia
OPENAI_API_KEY=你的OpenAI Key
WHISPER_MODEL=base
```

**获取 API Keys：**
- **Azure Speech**：[Azure 门户](https://portal.azure.com) → 创建"语音服务"资源 → 密钥和终结点
- **OpenAI**：[platform.openai.com](https://platform.openai.com) → API Keys

### 2. 启动系统

**Windows：**
```
双击 start.bat
```

**Linux / Mac：**
```bash
chmod +x start.sh
./start.sh
```

首次启动会自动安装所有依赖（约需 3-5 分钟）。

### 3. 访问

打开浏览器访问：**http://localhost:5173**

## 功能使用指南

### 一、初始水平测试（首次使用）

1. 点击导航栏右上角的水平标签，进入水平测试页面
2. 点击**开始测试**，系统播放 5 段由易到难的音频
3. 每段听完后在输入框中听写你听到的内容（可多次播放）
4. 全部完成后点击**提交并完成**
5. 系统自动评估你的初始水平（1-10分）

> 建议诚实作答，错误越多，初始水平越低，训练越从基础开始。

---

### 二、上传训练内容

#### 方式 A：上传文件

1. 点击导航栏**上传**
2. 拖拽或点击上传音频/视频文件（支持 MP3、MP4、WAV、M4A、MKV）
3. 点击**开始上传**
4. 系统自动处理（转录、切割、评分），通常需要 1-5 分钟

#### 方式 B：YouTube 链接

1. 点击**上传**页面
2. 粘贴 YouTube 链接（如 `https://www.youtube.com/watch?v=xxxxx`）
3. 点击**提交**
4. 系统自动下载并处理

> **推荐内容**：TED Talk、播客访谈、英语口语课程、美剧片段

---

### 三、训练卡片

处理完成后，进入**训练库**页面，系统只显示适合你当前水平的卡片。

#### 卡片列表

每张卡片显示：
- **难度总分**（颜色：绿色=简单，黄色=适中，红色=困难）
- **五维度难度条**（语速/音变/词汇/句法/音质）
- **卡片状态**（新卡片/学习中/待复习/已掌握）

点击卡片进入训练。

#### 训练流程（四层递进）

| 层次 | 内容 | 目的 |
|---|---|---|
| 第一层 | 只听音频，不看文字 | 纯听力输入 |
| 第二层 | 显示文字 + 语音现象标注 | 理解为什么听不懂 |
| 第三层 | 0.75x 慢速播放 | 感受发音节奏 |
| 第四层 | 正常速度 + 高亮标注 | 综合训练 |

**语音现象颜色说明：**
- 🟠 橙色底线 = 连读
- 🔵 蓝色底线 = 弱读
- 🔴 红色底线 = 同化
- 🟢 绿色底线 = 省略
- 🟣 紫色底线 = 闪音

---

### 四、掌握验证（四关卡）

通过四层训练后，进入掌握验证：

#### Gate 1 — 跟读验证（Shadowing）

1. 按住**🎤 按住跟读**按钮，跟着原音频大声模仿（包括连读弱读）
2. 松手后系统自动评估
3. 评分 ≥ 90 分算通过，**连续通过 3 次**才解锁下一关
4. 重点模仿：不要读书面形式，要模仿音频中实际的发音方式

#### Gate 2 — 泛化测试（Generalization）

1. 点击**生成测试句子**
2. 系统生成一个包含相同语音规律但完全不同的新句子，并播放
3. 听写你听到的内容
4. 系统评估是否正确（接受合理的口语简化形式）
5. **意义**：验证你学的是规律本身，而不是在背诵具体音频

#### Gate 3 — 压力测试（Stress Test）

1. 点击**开始压力测试**
2. 系统随机加入干扰：背景噪音 或 1.5倍速
3. 在干扰条件下听写
4. 准确率 ≥ 85% 通过
5. **意义**：验证处理已自动化，不占用额外认知资源

#### Gate 4 — SRS 时间检验

前三关通过后，卡片进入 SRS 复习队列：
- 1天后复习 → 通过 → 6天后 → 通过 → 间隔逐渐延长
- **90天后复习仍通过** → 标记为"已掌握"

> 每天在**训练库**选择**复习**模式，完成当天到期的卡片。

---

### 五、词汇仪表板

点击导航栏**词汇**进入词汇仪表板。

**颜色含义（贝叶斯掌握概率）：**
- 🔵 **蓝色** — P < 30%（未知词）
- 🟡 **黄色** — 30% ≤ P < 85%（学习中）
- ⬜ **白色** — P ≥ 85%（已掌握）

**概率更新规则：**
- 每次听写正确 → P 上升
- 每次听写错误 → P 下降
- 距上次复习时间越长 → P 自动衰减（遗忘曲线）
- 高难度条件下答对 → P 上升更多

可按颜色筛选词汇，查看各状态词汇数量和进度。

## 常见问题

**Q: 处理速度很慢？**
A: Whisper 转录是最耗时的步骤。建议使用 GPU（NVIDIA），或在 .env 中将 `WHISPER_MODEL=tiny` 减小模型。

**Q: Azure 评估失败？**
A: 检查 .env 中的 `AZURE_SPEECH_KEY` 和 `AZURE_SPEECH_REGION` 是否正确。免费账户有每月调用限制。

**Q: 语音现象标注不准确？**
A: 音素检测依赖 Azure Speech 识别质量。音频质量越好（清晰录音，无背景噪音），检测越准确。

**Q: 想重置水平怎么办？**
A: 点击导航栏右上角水平标签 → 重新测试，或在 API 文档（http://localhost:8000/docs）中手动调整。

**Q: 数据存储在哪里？**
A: 所有数据存储在 `data/` 目录：
- `data/db/listening.db` — 数据库
- `data/uploads/` — 原始文件
- `data/audio_segments/` — 切割后的片段

## 训练建议

1. **每天训练**：每次 15-30 分钟，集中注意力
2. **先完成复习**：每天先处理到期的复习卡片，再学新卡片
3. **跟读要真实**：Gate 1 不要"念文字"，要真正模仿音频的语流
4. **接受不完美**：水平测试或 Gate 评估失败是正常的，系统会自动调整
5. **坚持 i+1**：不要跳过筛选强行训练超出水平的内容，这会降低效率
