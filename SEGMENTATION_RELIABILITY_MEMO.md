# 听力切片可靠性优化备忘录

## 1. 背景与目标
当前系统切片链路已可用，但在小间隔句边界（如 80ms）和复杂口语场景下，仍会出现“句尾/句首被截断、文本与音频不一致”等坏切片问题。  
本备忘录目标：从“可用”升级到“可追溯、可回退、可重制、可持续优化”的生产级方案。

---

## 2. 核心改造方向（结论）

### A. LLM 输出“断点索引”，不再输出自由文本
- 从“生成句子文本再对齐”改为“基于 WhisperX token 序列直接返回切点索引”。
- 价值：显著降低 LLM 文本不稳定导致的对齐漂移。

### B. 两阶段决策：Stage A 规则候选 + Stage B LLM 受限选择
- Stage A（非 LLM）：根据词间停顿、句长、标点/连接词生成候选切点集合。
- Stage B（LLM）：只在候选集合中选择，不允许自由生成。
- 价值：把 LLM 从“生成器”降级为“排序器/选择器”，稳定性更高。

### C. 小间隔场景不采用均分边界
- 特别是间隔 <120ms 时，不采用机械中点切分。
- 改为“尾音优先 + 轻量重叠保留 + 波形裁决”。

### D. 波形辅助裁决切点
引入以下特征参与最终边界选择：
1. RMS 谷底（短时能量最小）
2. ZCR（过零率）
3. Spectral Flux（谱通量）
4. Voicing（有声/无声）

### E. 坏切片溯源（Lineage）
- 每个切片保留完整生成链路：候选点、LLM选择、波形评分、回退记录、最终参数。
- 在训练页可查看“该坏切片如何产生”的通俗时间线解释。

### F. 个别坏切片重制功能
- 支持单片重制（不重跑整条内容）。
- 支持策略切换（保守/激进）和参数覆盖。

---

## 3. 现状对照（当前实现）
- 目前切句流程：LLM 切句 → 词级匹配 → 边界中点调整。  
- 当前匹配：贪心 + 前3词锚点 + 首词回退。  
- 匹配失败时存在“文本拼接上一段”的策略，可能引发文本与音频边界不一致。

---

## 4. 目标架构（建议）

### 4.1 切片决策链
1) 候选切点生成（规则）  
2) LLM 受限选择（仅候选内选择）  
3) 波形裁决器二次优化（RMS/ZCR/Flux/Voicing）  
4) 小间隔保护（<120ms 启用重叠保留/尾音优先）  
5) 质量评分 + 自动回退（低分不放行）

### 4.2 溯源链
每个切片保存：
- 输入来源（content_id、segment_id、原音频范围）
- 候选切点及评分
- LLM 最终决策
- 波形微调前后边界
- 质量分、回退路径
- 用户反馈标签（切早/切晚/尾音缺失/句首缺失/语义不完整）

### 4.3 单片重制链
- 用户端提交“坏切片”标记。
- 后端触发单片重制任务。
- 重制后保留旧版本，支持 A/B 回听与回滚。

---

## 5. 上线节奏（建议）

### Phase 1（1~2周）
- LLM 返回断点索引
- Stage A 候选切点
- 单片重制 API

### Phase 2（2~4周）
- 波形裁决器接入
- 小间隔重叠保留
- 质量评分 + 自动回退

### Phase 3（持续）
- 溯源页面
- 管理端聚合看板
- 阈值自动调优

---

## 6. 质量指标（SLO）
- 坏切片放行率：< 0.5%
- 单片重制成功率：> 90%
- 用户报错后修复时效：P90 < 24h

---

## 7. 一句话总结
将切片系统从“单次结果产出”升级为“可约束决策 + 可解释溯源 + 可单片重制”的闭环工程系统。
